get_or_create_user(access_token)¶

Returns a User instance if 1 user is found. Creates a user if not found and configured to do so. Returns nothing if multiple users are matched.

get_token(payload)¶

Return token object as a dictionary. Borrowed from logindotgov-oidc, modified https://github.com/trussworks/logindotgov-oidc-py

get_userinfo(access_token)¶

Return user details dictionary. The id_token and payload are not used in the default implementation, but may be used when overriding this method

Return True if user_obj has any permissions in the given app_label.

raise_token_response_error(response)¶

Raises HTTPError, if one occurred. as per: https://datatracker.ietf.org/doc/html/rfc6749#section-5.2

update_user(user, claims)¶

user_can_authenticate(user)¶

Reject users with is_active=False. Custom user models that don’t have that attribute are allowed.

Validate the token signature.

with_perm(perm, is_active=True, include_superusers=True, obj=None)¶

Return users that have permission “perm”. By default, filter out inactive users and include superusers.
