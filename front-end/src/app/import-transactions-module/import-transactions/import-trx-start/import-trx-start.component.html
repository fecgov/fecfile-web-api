To Begin Importing

You can start your Import by taking clicking "Begin Upload" below, or by taking any of the 7 additional actions provided below::

Importing Files

  1. At the top right of the screen, select Tools and then Import Transactions. The Import Tranactions Landing Page (this page) will appear.
  2. Click "Begin Upload" to navigate to the Upload Page.
  3. Locate your import file using your computer's file explorer, and then drag and drop that file into the grey-shaded area provided, or use the "Upload" button option to open the file explorer and locate the file you would like to import. The import file should have a .CSV extension, or it wil not be accepted.
  4. Once the upload begins the file will be scanned for formatting errors. If there are any formatting errors the upload will be cancelled and an error message will be displayed listing all of the formatting errors found. If no errors are found you wil proceed to the "Review"page.
  5. Once at the "Review" page the file will be scanned for matching transactions within the file that may have been previously uploaded. Bascially scanning the import file to see if the file has previously been imported. If there are matching transactions from a previously uploaded file the system will provide a warning, and a prompt to proceed or not to prevent from possibly uploading the same file twice.
  6. Next the the system will scan for duplicate contact entries within the file against your contacts database. If there are possible duplicate contact entries, you will be taken to the "Clean" screen to select which contact entries you would like to keep or merge. You will also have the option to ignore the process allowing for duplicate entries from the uploaded file and your current contact database to exist. If no duplicates exist, the screen will display the text: "No Duplicates found."
  7. Next the Import will begin after the previous step. A loader with a completion percentage will be displayed allowing the user to view the progress of the import. Once the import is complete the user can navigate to the Notifications Modal (Top Menu/Notifications) to view the import and others via the link provided.

How to enter data into the CSV templates

When entering data into a CSV template, ensure the fields within the template match with what you are planning to import. CSV files will not always save information within the CSV format, so you may need to save the file as an excel document first, and then when complete save in a CSV format. Further Import Instructions and Information is provided below.

Additional Import Information and Instructions

  1. A Filer can only import transactions to the specific reports: F3X or F3L.
  2. Reports must be in a "Saved," or "Amended" status before starting the import process.
  3. Always clean the Contacts Database for possible duplicates before initiating an Import:
    • Filer can select one file or multiple files to drag & drop. The limit is 10 files. The Filer can only upload/import a .CSV file for one schedule. Transactions from different schedules cannot be combined into 1 CSV file.
    • Each CSV file must correspond to a schedule type and include transactions ONLY for that schedule.
    • The Filer can also split a schedule into separate files, but we recommend having all the corresponding transactions for each schedule in one file/schedule because of the limitation.
  4. Once all of the transactions are imported, the language required by FEC in certain fields of the transactions will be auto calculated.
    • Language: Some transactions will have language that is auto-generated. Please refer to the Form 3X format specifications to determine if you should leave the description field blank or if you need to include data.
  5. Aggregate amounts for each transaction will be auto calculated along with the existing transactions and displayed.
  6. Any transactions that will be auto generated based on imported transactions, will be auto generated and saved to the database. In-Kind Receipt (imported) → In-Kind Out must be auto generated.
  7. The Filer has to manually add the "Levin Account name" under "Profile" menu. When the filer is adding LAs or LBs into the template, they will need to add the "Levin Account name" (from the application) into that column.

Receipt Description & Purpose of Disbursement fields

Filers must review the "Value Reference" in the import specs for the exact language that should be included for "Receipt Description," and "Purpose of Disbursement Fields."

Employer & Occupation

Employer and occupation fields are required if an individual donor aggregate is greater than $200. Please note that the aggregate is auto-calculated by the system once importing is complete. If you anticipate that the the donor aggregate will be greater than $200 please include employer and occupation as part of the import file.

Auto-generate transactions

When an In-Kind transactions in imported, In-Kind Out will be auto-generated.

Loans and Debt transactions

  1. Loans & debt transaction must be manually entered into the system.
  2. Loan & debt payment transactions must be manually entered into the system.

Bundled Transactions and/or Auto-Generated Transactions

  1. Once all the transactions are imported, the language required by FEC in certain fields of the transactions must be auto populated.
    • Some transactions will have language that is auto-generated. Please refer to the Form 3X format specifications to determine if you should leave the description field blank or if you need to include data.
  2. Aggregate amounts for each transaction must be auto calculated along with the existing transactions and displayed.
  3. Any transactions that need to be auto generated based on imported transactions, must be auto generated and saved to te database. In-Kind Receipt (imported) → In-Kind Out must be auto generated.

Importing H3 - H6 transactions

Make sure H1 & H2 ratios exist or are manually added before importing H3 - H6 transactions.
Event Types
  1. Administrative – H3, H4 = AD
  2. Direct Fundraising – H3, H4 = DF
  3. Exempt Activity – H3, H4 = EA
  4. Generic Voter Drive – H3, H5 = GV
  5. Direct Candidate Support – H3, H5 = DC
  6. Public Communications – H3, H5 = PC
  7. Voter Registration – H5, H6 = VR
  8. GOTV – H5, H6 = GOTV
  9. Voter ID – H5, H6 = VI
  10. Generic Campaign – H5, H6 = GC
  1. If a H2 event is selected, Activity or Event Identifier column in the template must be entered with the unique H2 name that the filer has created in the application.
  2. Account name for H3 & H5, LA, LB:
    1. H3 - filer adds a name.
    2. For H5, LA, LB - Filer must first manually add the Levin account to the application via Profile menu. Filer must then add that name here for any H5, LA or LB transactions.

Importing Earmarks

All Conduit Earmark transactions for Deposited & Undeposited must be manually entered into the system.
